隐私政策

最后更新：2026 年 7 月 13 日

一句话总结

PiPanel 不收集、不上传、不与任何人分享你的使用数据。核心画中画功能完全本地运行，没有账号系统、没有统计分析、没有崩溃上报。唯一会联网的场景是购买和验证付费授权，这部分数据由第三方支付处理商 Creem 处理，见下方「购买与授权验证」章节。核心功能的代码公开托管在 GitHub 上，可以自行验证。

App 需要哪些系统权限，用来做什么

PiPanel 在首次使用时会请求以下两项 macOS 系统权限，两者都只在你的 Mac 本地生效，不会把任何内容发送到外部服务器：

  • 屏幕录制（Screen Recording）：用于通过 ScreenCaptureKit 捕获你选中窗口的画面，并在画中画悬浮窗里实时显示。捕获到的画面只在本机内存中处理和渲染，不会被保存成文件，也不会被上传。
  • 辅助功能（Accessibility）：用于读取/调整源窗口的位置和尺寸，并把你在画中画悬浮窗里的点击、滚动、按键操作转发到源窗口上。

本地存储的设置

PiPanel 会把你的偏好设置（例如画面帧率、默认尺寸、外观选项等）保存在 macOS 标准的本地偏好设置（UserDefaults）中，仅存在于你自己的 Mac 上，不会同步到任何账号或云端。

网络请求

PiPanel 的开源版本（公开在 GitHub 上的核心代码）不发起任何网络请求，没有统计分析（analytics）、没有崩溃报告工具、没有广告 SDK，也没有任何形式的遥测（telemetry）。正式发布的 App 额外包含一个未开源的付费授权模块，只在你主动购买或激活授权时才会联网，具体见下一节。

购买与授权验证

如果你购买了付费授权，PiPanel 会在以下场景联网：

  • 购买时：点击购买会跳转到第三方支付与授权处理商 Creem 的结算页面完成付款。你的支付信息（卡号、账单地址等）由 Creem 直接收集和处理，PiPanel 和开发者都不会接触到这些信息，具体以 Creem 自身的隐私政策为准。
  • 激活/验证授权码时：App 会把你输入的授权码和一个本机生成的设备标识符发送给 Creem 的服务器，用来确认授权有效并绑定到你的设备。设备标识符和授权码保存在 macOS 的 Keychain 中，仅用于这个用途。

除上述两个场景外，PiPanel 不会发起其他任何网络请求。

关于这个网站

本网站（lyle-xub.github.io/PiPanel）托管在 GitHub Pages 上，本身不包含任何统计分析脚本。GitHub 作为托管方可能会按照其自身的隐私政策记录基础的访问日志（例如 IP 地址），这部分由 GitHub 官方隐私声明约束，与开发者无关。

联系方式

如果你对本隐私政策有任何疑问，欢迎在 GitHub Issues 中提出。
